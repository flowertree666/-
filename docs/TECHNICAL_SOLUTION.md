# 技术方案说明

## 总体架构

Vue 3 负责需求采集与多方案展示；Spring Boot 提供景点查询、画像解析、客流评级和路线规划 API；MySQL 保存景点快照及来源元数据；高德地图 Web 服务用于生产环境的距离矩阵与路径时间。

数据链路采用“采集适配器 → 清洗去重 → 可信度评分 → 动态景点库 → 规划引擎”。`AttractionDataProvider` 是数据源扩展接口，定时任务可按城市拉取获授权平台、景区官网、文旅开放数据。每条记录必须保留来源链接、采集时间和置信度。

## 规划策略

1. 偏好匹配：标签及分类命中加权。
2. 时令适配：当前月份/季节适配加权。
3. 热度与客流：热门方案强调热度；慢游方案加大拥挤惩罚。
4. 空间聚类：先选当日锚点，再按地理距离安排邻近景点，减少折返。
5. 时间约束：景点建议时长 + 转场缓冲不得超过每日预算。
6. 多目标输出：经典均衡、深度慢游、热门打卡三套方案。

当前 MVP 使用 Haversine 距离稳定降级。生产接入高德距离测量/步行/公交路径规划后，应将实际转场时间写入约束计算，并缓存相同坐标对结果以控制配额。

## 客流预测

建议生产模型特征包括平台近期热度、预约余量、节假日、星期、天气、小时、历史客流和活动事件，输出 0—100 客流指数、趋势与置信区间。界面必须展示更新时间和数据来源；信息过期时降级为“待刷新”，绝不将预测包装成官方实时客流。

## 安全与合规

- 遵守 robots.txt、平台服务条款、著作权和个人信息保护要求。
- 不绕过登录、验证码、签名或访问控制；优先官方开放 API 和授权供应商。
- API Key 只存后端环境变量，设置访问白名单、限流、缓存和审计。
- 用户画像遵循最小化存储，可匿名使用并支持删除。
