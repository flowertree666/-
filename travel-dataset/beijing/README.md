# 北京故宫博物院 POI 数据集

首版采集日期：2026-07-16

## 文件

- `pois.jsonl`：1 条故宫博物院主 POI 记录。
- `knowledge_chunks.jsonl`：6 条可用于 RAG 的知识片段。
- `sources.jsonl`：7 条来源记录，含官方静态规则、2026 年动态公告和地图坐标来源。
- `itinerary_cases.jsonl`：3 条可判定的路线评测样例。
- `rag_eval_cases.jsonl`：10 条 RAG 问答、动态过滤和不可回答问题评测样例。
- `claim_evidence.jsonl`：14 条事实级结论与来源映射，用于引用生成和审计。

## 数据边界

本数据集只将“故宫博物院”作为主 POI。天安门广场、景山公园、北海公园、前门大街等作为相邻景点，不并入故宫主 POI。珍宝馆、钟表馆和临时展览的票务/预约信息作为故宫内部专馆或动态证据处理，不另建主 POI。

## 口径与注意事项

1. 坐标采用 GCJ-02，经度在前；坐标是故宫博物院中心点，不等同于午门入口点。
2. 开放时间、票价、预约和参观路线均优先采用故宫博物院官网及官方订票页。
3. 动态公告包含 `valid_until`，超过有效期后不得继续作为近期推荐依据。
4. 页面未提供明确发布日期的长期规则使用 `published_at: null`，但保留 `collected_at` 以便复核。
5. JSONL 一行一个 JSON 对象；`source_ids` 用于将 POI 和知识片段追溯到来源。
6. `content_type` 区分静态介绍、票务规则、交通、游客服务、路线建议和动态公告。
7. `review_after` 用于静态规则的定期复核；`valid_until` 用于有明确结束时间的动态证据。

## RAG 使用建议

采用“关键词检索 + 向量检索 + `poi_id` 过滤 + `valid_until` 过滤 + 官方来源优先”的混合检索。实时余票、当天开放状态和临时展览不可由本数据集直接推断，应引导用户查看官方订票页或最新公告。

## 首轮复核建议

使用本数据进行线上推荐前，重新核对故宫官方订票页的预约日期、票价、临时闭馆和特展信息；尤其要复核法定节假日开放安排以及动态公告是否已过期。
